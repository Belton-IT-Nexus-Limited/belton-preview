Image Treatment Guide

Seven approaches that preserve the dark aesthetic. Copy these patterns when adding images to the site.

1. Atmospheric Background

Very dark, desaturated image as subtle texture. Barely visible but adds depth and visual interest without competing with content.

The image sits at 7% opacity with full grayscale, creating atmosphere without competing with content. Works best with busy images that become abstract patterns when darkened.

When to use

  • Hero sections that need subtle depth
  • Feature highlight areas
  • CTA sections where you want atmosphere
  • Best with: tech imagery, office shots, abstract patterns
.atmosphere-bg {
    position: absolute;
    inset: 0;
    background-image: url('image.webp');
    background-size: cover;
    opacity: 0.07;
    filter: grayscale(100%) contrast(1.2);
}

2. Contained Image with Glow

Image in a container with subtle red glow border. Slightly desaturated and darkened to fit the theme. Feels premium and on-brand.

When to use

  • Standalone feature images
  • Service page hero images
  • Case study imagery
  • Key values: brightness(0.8), saturate(0.9)
  • Glow: rgba(230, 57, 70, 0.15) for subtle brand accent
.contained-image::after {
    border: 1px solid rgba(230, 57, 70, 0.3);
    box-shadow:
        0 0 40px rgba(230, 57, 70, 0.15),
        inset 0 0 60px rgba(0, 0, 0, 0.4);
}

3. Split Content Layout

Text on one side, image on other with gradient fade into the background. Creates visual interest while keeping content readable.

Security operations that never sleep

Our team monitors your environment around the clock. When threats emerge, we respond in minutes, not hours. Real people, real expertise, real protection.

When to use

  • Feature highlights on service pages
  • About page sections
  • Breaking up long prose content
  • Gradient fades image into var(--bg)
  • Image filter: brightness(0.7) saturate(0.8)
  • Can flip layout (image left, text right) for variety

3b. Split Content with Hover Reveal

Image heavily faded by default, reveals to full visibility on hover. Creates intrigue and rewards interaction. Hover over the section below.

Real people. Real expertise.

Not a call centre overseas. Not a chatbot. When you need help, you get our Auckland team on the line. People who know your systems and can actually fix things.

When to use

  • Interactive feature sections
  • When you want the image to feel like a reward
  • Sections that link to more detail (case studies, about)
  • Default: brightness(0.4) saturate(0.5) grayscale(40%)
  • Hover: brightness(0.85) saturate(0.95) grayscale(0%)
  • Subtle scale on hover: transform: scale(1.02)
  • Text shifts slightly on hover for polish
.split-hover .split-image img {
    filter: brightness(0.4) saturate(0.5) grayscale(40%);
    transition: filter 0.5s ease, transform 0.5s ease;
}

.split-hover:hover .split-image img {
    filter: brightness(0.85) saturate(0.95) grayscale(0%);
    transform: scale(1.02);
}

5. Accent Strip

Thin image band as section divider. Heavily darkened with gradient fades top and bottom so it blends seamlessly.

When to use

  • Between major page sections
  • Before CTA sections
  • To add visual rhythm on long pages
  • Height: 120px works well (80px-150px range)
  • Image filter: brightness(0.4) saturate(0.7)
  • Gradient fades prevent hard edges
  • Best with: wide landscape images, tech/office imagery

6. Duotone / Brand Tinted

Fully grayscale image with subtle red tint overlay. Creates brand cohesion and artistic feel.

When to use

  • Hero backgrounds
  • Feature imagery where brand cohesion matters
  • When full-color would be too distracting
  • Base: grayscale(100%) contrast(1.1) brightness(0.8)
  • Overlay: rgba(230, 57, 70, 0.2) with mix-blend-mode: overlay
  • Adjust red intensity (0.1-0.3) based on image

7. Image Card

Image as part of a content card. Compact, contained, pairs image with related text content.

Auckland-based support

Real people in a real office. When you call, someone answers. When you need help, we're there.

When to use

  • Case studies and success stories
  • Team member highlights
  • Location/office information
  • News or announcement cards
  • Image width: 200px (adjust 150px-250px as needed)
  • Image filter: brightness(0.85)
  • Stacks vertically on mobile

General Guidelines

Image treatment principles

  • Always darken - Use brightness(0.7-0.9) on all images
  • Desaturate slightly - saturate(0.7-0.9) keeps colors muted
  • Use gradients - Fade images into var(--bg) for seamless blending
  • Brand accent sparingly - Red glow/tint at low opacity (0.1-0.2)
  • Prefer WebP - Smaller file sizes, same quality
  • Responsive images - Provide 800w and 1200w versions for srcset
  • Object-fit: cover - Ensures images fill containers without distortion

What to avoid

  • Full-brightness, fully-saturated images
  • Hard edges where image meets background
  • Stock photo cliches (handshakes, pointing at screens)
  • Images that compete with text for attention
  • Decorative images that add no meaning
  • Hero images without treatment (too bright/distracting)
